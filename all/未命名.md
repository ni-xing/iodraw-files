个人博客系统iodraw类图代码

说明：代码可直接复制到iodraw中生成类图，覆盖需求分析中所有核心模块（用户、文章、分类、标签、互动），关联关系贴合业务逻辑，属性和方法简化适配新手开发，无复杂冗余内容。

// 个人博客系统iodraw类图代码（直接复制可用）
@startuml
' 核心实体类
class User {
  - id: Long          // 主键ID
  - username: String  // 登录账号
  - password: String  // 加密存储的密码
  - nickname: String  // 昵称
  - avatar: String    // 头像路径（本地）
  - intro: String     // 个人简介
  - createTime: Date  // 创建时间
  + login(username: String, password: String): Boolean  // 登录方法
  + updateInfo(nickname: String, avatar: String, intro: String): void  // 修改个人信息
  + getInfo(): User   // 获取个人信息
}

class Category {
  - id: Long          // 主键ID
  - name: String      // 分类名称
  - createTime: Date  // 创建时间
  + addCategory(name: String): void  // 添加分类
  + updateCategory(id: Long, name: String): void  // 编辑分类
  + deleteCategory(id: Long): void  // 删除分类
  + getCategoryList(): List<Category>  // 获取所有分类
}

class Tag {
  - id: Long          // 主键ID
  - name: String      // 标签名称
  - createTime: Date  // 创建时间
  + addTag(name: String): void  // 添加标签
  + updateTag(id: Long, name: String): void  // 编辑标签
  + deleteTag(id: Long): void  // 删除标签
  + getTagList(): List<Tag>  // 获取所有标签
}

class Article {
  - id: Long          // 主键ID
  - title: String     // 文章标题
  - content: String   // 文章内容（Markdown/HTML格式，text类型）
  - categoryId: Long  // 关联分类ID
  - tagIds: String    // 关联标签ID，用逗号分隔
  - status: Integer   // 状态：0-草稿，1-已发布
  - likeCount: Integer  // 点赞数
  - commentCount: Integer  // 评论数
  - createTime: Date  // 发布/创建时间
  - updateTime: Date  // 更新时间
  + addArticle(article: Article): void  // 发布/保存草稿
  + updateArticle(article: Article): void  // 编辑文章
  + deleteArticle(id: Long): void  // 逻辑删除文章
  + getArticleById(id: Long): Article  // 根据ID获取文章
  + getArticleList(status: Integer, categoryId: Long, tagId: Long): List<Article>  // 获取文章列表（带筛选）
  + searchArticle(keyword: String): List<Article>  // 搜索文章
}

class Comment {
  - id: Long          // 主键ID
  - articleId: Long   // 关联文章ID
  - nickname: String  // 评论者昵称（匿名）
  - content: String   // 评论内容
  - parentId: Long    // 父评论ID（用于回复，0表示一级评论）
  - createTime: Date  // 评论时间
  + addComment(comment: Comment): void  // 添加评论
  + replyComment(comment: Comment): void  // 回复评论
  + deleteComment(id: Long): void  // 删除评论
  + getCommentByArticleId(articleId: Long): List<Comment>  // 根据文章ID获取评论
}

class Like {
  - id: Long          // 主键ID
  - articleId: Long   // 关联文章ID
  - visitorKey: String  // 访客标识（浏览器缓存，无需登录）
  - createTime: Date  // 点赞时间
  + addLike(articleId: Long, visitorKey: String): Boolean  // 点赞（判断是否已点赞）
  + cancelLike(articleId: Long, visitorKey: String): void  // 取消点赞
  + getLikeCount(articleId: Long): Integer  // 获取文章点赞数
  + isLiked(articleId: Long, visitorKey: String): Boolean  // 判断是否已点赞
}

class DataStatistic {
  - articleTotal: Integer  // 文章总数
  - publishedTotal: Integer  // 已发布文章数
  - draftTotal: Integer  // 草稿数
  - commentTotal: Integer  // 总评论数
  - likeTotal: Integer  // 总点赞数
  - viewCount: Integer  // 访问量
  + getStatisticData(): DataStatistic  // 获取统计数据
}

' 类之间的关联关系
User "1" -- "0..*" Article : 拥有（作者发布多篇文章）
Category "1" -- "0..*" Article : 包含（一个分类有多篇文章）
Tag "1" -- "0..*" Article : 关联（一个标签关联多篇文章）
Article "1" -- "0..*" Comment : 拥有（一篇文章有多个评论）
Article "1" -- "0..*" Like : 拥有（一篇文章有多个点赞）
Comment "1" -- "0..*" Comment : 回复（一个评论可有多条回复）
User "1" -- "1" DataStatistic : 查看（作者查看自己的统计数据）

@enduml

补充说明：

- 类的属性和方法完全贴合需求分析，无多余功能，适配SpringBoot + MyBatis-Plus开发，可直接对应实体类编写。

- 关联关系说明：1表示一对一/一对多中的“一”，0..*表示“零个或多个”，贴合实际业务（如一个作者可发布多篇文章，一篇文章可关联多个标签）。

- 复制代码后，打开iodraw，创建“UML类图”，粘贴代码即可自动生成可视化类图，可直接修改调整。
